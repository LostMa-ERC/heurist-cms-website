Several record types contain temporal information, especially about when an object began to exist. For example, the Text record type requires a date_of_creation. Dates in Heurist can be precise or imprecise.

The type of metadata you can attach to an imprecise date depends on how you are entering data.

CSV import

When entering data via CSV import, you can only provide a precise date or a simple date range. The granularity of the precise date or the beginning and ending of the date range can vary. You can give a year (YYYY), a month (YYYY-MM), or a day (YYYY-MM-DD).

To provide a range, separate the two dates by a period. In the example shown below, we see a column containing a precise date, the year 1180, and a range, between 1230 and 1235.

date_of_creation
1180
1230.1235

These are the only types of information that can be provided via CSV import.

Manual import

Heurist's user interface, however, allows for more precision about the imprecision of a date or a date range. By clicking the range icon next to the data field, you open a pop-up window that has four types of date formats: Simple Date, Simple Range, Fuzzy Range, and Radiometric. In our workflow, you are free to use any of the first three, each of which is described in detail below.

Simple Date

The Simple Date tab allows you to attach metadata to a year, month, or day.

After entering the precise date, select one of the radio buttons to the right: "Exact," "Circa/approximate," "Before," or "After." Heurist will automatically select, as a default, the button "Exact."

In our workflow, we do not expect you to describe the type of certainty about the simple date, as indicated by the four levels of "Type of Determination": unknown, attested, conjecture, and measurement. You can enter this information if you want, but it will not be present on my records and no scaled treatment of this information is planned.

Do not adjust the type of calendar used on the Simple Date. Leave the default Gregorian selection.

In summary:

  • Necessary:
    • Date (Year or ISO date)
    • Metadata about date (Exact, Circa/approximate, Before, After)

Simple Range

The Simple Range tab allows you to attach metadata to a set of two dates. The two dates indicate a simple span of time.

In our workflow, we do not expect you to add any further information about the certainty of the range. To go into further detail, we recommend you use the Fuzzy Range tab.

In summary:

  • Necessary:
    • Earliest Date (Year or ISO date)
    • Latest Date (Year or ISO date)

Fuzzy Range

The Fuzzy Range tab allows you to detail two overlapping spans of time, the possible and the probable. The fuzzy range is based on four dates:

  • Terminus post quem: The earliest point in time when it would be possible for the event to have taken place.
  • Probable begin: The earliest point in time when it is likely the event took place. To set it to the same as the terminus post quem, click the "set" button.
  • Probable end: The latest point in time when it is likely that the event took place. To set it to the same as the terminus ante quem, click the "set" button.
  • Terminus ante quem: The latest point in time when it would be possible for the event to have taken place.

From the drop-down menus "Start Profile" and "End Profile," you can detail the distribution of probability between the possible extreme time points and the likely internal time points. The options are the following: flat, central, slow start, slow finish. When you do not know the distribution, leave the default value: flat.

  • Necessary:
    • Terminus post quem (Year or ISO date)
    • Probable begin (Year or ISO date)
    • Probable end (Year or ISO date)
    • Terminus ante quem (Year or ISO date)
    • Start profile (flat, central, slow start, slow finish)
    • End profile (flat, central, slow start, slow finish)
